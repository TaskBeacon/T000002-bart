# Parameter Mapping

## Mapping Table

| Parameter ID | Config Path | Implemented Value | Source Paper ID | Evidence (quote/figure/table) | Decision Type | Notes |
|---|---|---|---|---|---|---|
| `trial_count` | `task.total_trials`, `task.trial_per_block` | Human `10` total (`2 x 5`), QA/sim `9` total | `W2146196757` | Canonical BART protocol is commonly instantiated with 30 balloons. | `adapted` | Current trial count is substantially reduced relative to canonical sampling depth. |
| `condition_set` | `task.conditions` | `['blue', 'yellow', 'orange']` | `W2005240523` | Modified BART variants use color-coded balloon types with distinct burst distributions. | `adapted` | Current color set is three-level risk coding; `orange` is used where many protocols use `red`. |
| `max_pumps_profile` | `task.blue_max_pumps`, `task.yellow_max_pumps`, `task.orange_max_pumps` | `24`, `12`, `6` | `W2005240523` | fMRI modified BART describes color-specific burst ranges (example: blue/yellow/red with wider ranges). | `adapted` | Current caps appear down-scaled from common 128/32/8 style variants and need explicit protocol justification. |
| `reward_increment` | `task.blue_delta`, `task.yellow_delta`, `task.orange_delta` | `5`, `10`, `20` points per successful pump | `W2005240523` | Modified BART examples often increment temporary account by fixed points per successful pump. | `adapted` | Current implementation uses condition-dependent delta, which changes risk-reward structure. |
| `action_mapping` | `task.pump_key`, `task.cash_key`, `task.key_list` | pump=`space`, cash=`right` | `W2146196757` | Core action pair is inflate-versus-collect (continue risk vs secure earnings). | `inferred` | Specific keyboard keys are implementation choices and should remain config-defined. |
| `response_deadline` | `timing.balloon_duration` | `2.0s` | `W2146196757` | Canonical BART is typically self-paced rather than a strict per-decision timeout window. | `adapted` | 2-second deadline introduces omission behavior not standard in original BART. |
| `explosion_sampling_method` | `src/run_trial.py::random.randint(1, max_pumps)` | Independent with-replacement draw each trial | `W2005240523` | Modified protocol describes burst points sampled without replacement from bounded integer sets per balloon color. | `adapted` | Current generator does not enforce without-replacement sampling pools. |
| `explosion_reproducibility` | `task.seed_mode` + runtime random source | Block condition seeds set, but explosion draws use global RNG in `run_trial.py` | `W2146196757` | Trial-level burst logic should be reproducible for auditable behavioral analyses. | `inferred` | Explosion sequence is not deterministically tied to block/trial seed in current code path. |
| `score_bank_initialization` | `src/run_trial.py::score_bank = <condition>_delta` | Starts at positive delta before first pump | `W2146196757` | Pumping is the operation that increases temporary trial earnings before cash-out. | `inferred` | Current logic permits positive cash-out without any successful pump; should be reviewed. |
| `primary_metric_support` | `main.py` output schema | Per-trial outcomes saved; adjusted-pumps summary not computed | `W2029537564` | BART validation literature emphasizes adjusted pump metrics on unexploded balloons. | `inferred` | Recommend adding analysis-ready adjusted-pumps fields or post-hoc script contract. |
